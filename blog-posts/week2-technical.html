Classes and IDs

By Nelly Lam

Sunday, July 27, 2014

Why would you use a class or id?

When designing a webpage, we use the language of html. Built into the language are elements such as header, main, section, aside etc. Each of those help the web designer layout their webpage, but they only go so far. As creative people tend to break the mold, it's easy to find a situation where the given elements will not accomplish the styling that one needs.

Scenario A:

You want to create a container for all your blog posts on one page so you can add a blue background behind your articles, but there is no specific html element for that. You could use a div, but you have other div elements already, and you don't want to make all of those blue too.

Scenario B:

You want to highlight a quote from your article by styling it differently from all the other paragraph text. You may have to use more quotes later in the article too.

This is where classes and ids come in handy.

What are classes and ids?

Classes and ids are 'hooks' that can be attached to a particular element and then be 'selected' in the stylesheet (the CSS file).

How are they used?

To create a class or id, you just have to define it inside the element tag.

In html:

	
	<div class='quotes'>
	<div id='blogwrapper'>

In css:

	
	.quotes{
		width: 600px;
	}
	#blogwrapper{
		width: 300px;
	}

The only difference when selecting the element in the css is that classes are called by using . and ids are called by using # in front of their names.

What is the difference between a class or an id?

The only difference between a class and an id is that classes can be used more than once on a page, and an id once and only once.

A class is like a common characteristic or a shared attribute. When you think of a class of students, you think many. Or economy class on an airplane, which contains many seats.

An id is unique. If I'm a teacher and I take attendence by calling out a student, I would call out one unique id.

When should I use a class, and when should I use an id?

Ask yourself, will I be using this more than once? If so, make a class. If not, make an id.

Now see if you can figure out which ones to use for Scenario A and B above.
