How Things are Stored

Arrays and hashes are both ordered collections.

An array is an ordered collection of objects. Since anything in Ruby is an object, anything can be stored as an element in an array. The way an array organizes the elements is by a zero-based index.

A hash is an ordered collection of keyed values. Each element in a hash is called a key-value pair. The order is based on the insertion order, the order in which the key element is created.

When to use an Array vs a Hash

We always come across the question of how to best collect and organize our data. To decide which one to use, one must first understand what is the type of data that is being stored, and how will it be manipulated, and how will it be accessed?

Is the type of data a really long list that must stay in a particular order? For example, you would probably want to keep the order of a sentence string that has been split into an array of letters. If it does not need to stay in a particular order, such as a set of key-value pairs such as name: 'Nelly' and age: 28, then a hash is probably the better option.

How will you be manipulating the data once it is stored? The Array in Ruby includes an extensive list of methods that you can use to manipulate it. Hashes also come with many methods to manipulate them, but the list is more limited.

How you plan to access the elements in your collection may be the deciding factor. In a hash, you can lookup a value based on the key, or vice versa. In an array, you lookup an element based on the index. It is easier to access a value based on a key, because usually the key is a variable name, and the order is not important. For example, hash_nelly[:age] => 28. You can only access elements in an array if you know exactly which index it is in: sentence_array[0] = 'a'.

I find that arrays are often used when I need to iterate through something in a specific order, add or delete the first or last elements, or if I need to iterate through standalone elments.

Hashes come in handy when there are key-value pairs or associative data that must stick together, or when a list of data belongs to a variable, almost like when working with classes. The best example is that of a person, which has many attributes such as name, age, sex, favorite colors, etc. Sometimes a hash is used instead of creating a new class. It is also easy to picture this if you have worked with JavaScript object literals.
